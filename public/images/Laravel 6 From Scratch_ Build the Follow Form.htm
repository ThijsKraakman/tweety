About This Episode

Let's build a "Follow Me" form for the profile page. This should toggle the follow status for the given user. To implement this, we'll discuss a few different approaches that you might consider.

    • it appears that the routing logic for {user:name} used on the 'profile' route (at 3:33) is only effective when one's explicitly passing the parameters that follow that uri format (e.g. /profiles/johndoe). Since Jeffrey is passing $auth->user (at 12:00), with getRouteKeyName() removed from the model, Laravel is defaulting to the usual Route Model Binding. As of now, it appears like you have to have to be explicit when passing data to that route for the new routing syntax to work...or create a workaround, as Jeffrey later does.

    Another solution like @jeffreyway hinted for the bottom border on the tweet loop, is if you're using Tailwind CSS is to set last:border-b-0.

    Just be aware you may need to set it in your tailwind.config.js as a variant like I found out when trying to use that the other day. See: Tailwind Documentation

    tailwind.config.js

    module.exports = {
  variants: {
    borderWidth: ['responsive', 'last', 'hover', 'focus'], 
    // Plus any other variants you want
  },
